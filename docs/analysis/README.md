## Analiz dokümanları

Amaç: Webde olan yetenekleri mobilde sunmak için **“ne istiyoruz?”** sorusunu netleştirmek.

### Kural

- Analizde **nasıl yapılacağı** yazılmaz (teknik tasarım/implementasyon detayı yok).
- Sadece **kapsam**, **kabul kriteri**, **iş kuralları**, **hata durumları** yazılır.

### Şablonlar

- `TEMPLATE-epic.md`
- `TEMPLATE-feature.md`

### Klasörler

- `epics/`: Epic dokümanları
- `features/`: Feature dokümanları
- `screens/`: Ekran bazlı gereksinimler (opsiyonel)
